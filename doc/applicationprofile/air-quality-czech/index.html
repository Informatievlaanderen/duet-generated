This document describes an application profile, in this case Czech Air Quality. The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

This application profile corresponds to the air quality data disclosed in the Czech airquality sensors

Status of this document

This application profile has the status Ontwerp Standaard publised at 2021-05-01.

Information about the process and the decisions involved in the creation of this specification are consultable at the registry of standards.

License

This specification of ORGANISATION is published under "CC0".

Conformiteit

An implementation is conformant with this application profile if it satifies these rules.

Overview

This document describes the usage of the following entities for a correct usage of the Application Profile:
| Component | Component Type | Region | State | Station |

Entities

Component

Description
A measurement for an air quality component
Properties
For this entity the following properties are defined: code, interval, value.
Property Expected Range Cardinality Description Usage Codelist
code Component Type the air quality component that is measured
interval String 1 the interval between measurements
value 1 the value measured in the interval

Component Type

Description
the kind of air quality component
Properties
For this entity the following properties are defined: code, name, unit.
Property Expected Range Cardinality Description Usage Codelist
code String 1 a notation identifying the air quality component
name String 1 a unique short label given to an entity
unit String 1 the unit of measurement for the air quality component

Region

Description
a subdivision of the state
Properties
For this entity the following properties are defined: code, name, station.
Property Expected Range Cardinality Description Usage Codelist
code String 1 a notiation identifying the entity
name String 1 a unique short label given to an entity
station Station a air quality sensor station located in the region

State

Description
A country
Properties
For this entity the following properties are defined: code, date from (UTC), date from-to, date to (UTC), name, region.
Property Expected Range Cardinality Description Usage Codelist
code String 1 a notation identifying the entity
date from (UTC) String 1 the measurement period start datetime in UTC the value could be according to xsd:datetime but currently are not
date from-to String 1 the measurement period in the default timezone of the Czech republic A non standard notation combining start and end date time into one serialized value
date to (UTC) String 1 the measurement period end datetime in UTC the value could be according to xsd:datetime but currently are not
name String 1 a unique short label given to an entity
region Region a region of the state

Station

Description
an air quality sensor station
Properties
For this entity the following properties are defined: classification, code, component, latitude, longitude, name, owner.
Property Expected Range Cardinality Description Usage Codelist
classification String 1 a classification of the station
code String 1 a notation identifying the entity
component Component a measurement for an air quality component by the station
latitude 1 the latitude position of the station The CRS is wgs84
longitude 1 the longituge position of the station The CRS is wgs84
name String 1 a unique short label given to an entity
owner String 1 the agent that is responsible for the station
