Dieses Dokument beschreibt ein Anwendungsprofil, in diesem Fall Demo-De. Das Anwendungsprofil spezifiziert ein semantisches Datenmodell, welches eine Reihe von Anwendungsfällen abdeckt. Die Spezifikation besteht aus Begriffen mit ihren Einschränkungen (Kardinalitäten, Codelisten, Nutzungsrichtlinien).

Übersicht

In diesem Dokument wird die Verwendung der folgenden Entitäten für eine korrekte Verwendung des Anwendungsprofils beschrieben:
| Adresse | Stadt |

Entitäten

Adresse

Beschreibung
die Einzelheiten des Ortes, an dem jemand lebt oder an dem sich eine Organisation befindet.
Eigenschaften
Für diese Entität wurden die folgenden Eigenschaften definiert: hausnummer, postleitzahl, stadt, strassenname.
Eigenschaft Erwarteter Datentyp Kardinalität Beschreibung Verwendung Codeliste
hausnummer String 1 Eine eindeutige Angabe, normalerweise eine Zahl, in einer Straße, die Orten zugeordnet ist, meistens Häusern oder Einheiten innerhalb von Häusern
postleitzahl String 1 Eine Gruppe von Zahlen oder Buchstaben und Zahlen, die einer Postanschrift hinzugefügt werden, um das Sortieren von Post zu erleichtern.
stadt Stadt 0..1 Die Stadt in der sich die Adressen befinden.
strassenname String 1 Der Name der öffentlichen Straße in einer Stadt oder einem Dorf entlang der Adresse befindet sich

Stadt

Beschreibung
eine menschliche Siedlung
Eigenschaften
Für diese Entität wurden die folgenden Eigenschaften definiert: name.
Eigenschaft Erwarteter Datentyp Kardinalität Beschreibung Verwendung Codeliste
name String 1 der Name der Stadt
