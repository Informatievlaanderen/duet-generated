This document describes an application profile, in this case Demo-En. The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

Status of this document

This application profile has the status Ontwerp Standaard publised at TBD.

Information about the process and the decisions involved in the creation of this specification are consultable at the registry of standards.

Conformiteit

Overview

This document describes the usage of the following entities for a correct usage of the Application Profile:
| Address | City |

Entities

Address

Description
the particulars of the place where someone lives or an organization is situated.
Properties
For this entity the following properties are defined: city, house number, post code, street name.
Property Expected Range Cardinality Description Usage Codelist
city City 0..1 The city in which the address is located
house number String 1 An unique indication, usually a number, in a street allotted to places, mostly houses or units within houses
post code String 1 A group of numbers or letters and numbers which are added to a postal address to assist the sorting of mail.
street name String 1 the name of the public road in a city, town, or village along the adress is located

City

Description
a human settlement
Properties
For this entity the following properties are defined: name.
Property Expected Range Cardinality Description Usage Codelist
name String 1 the name of the city
